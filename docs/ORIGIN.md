# Origin

This repo exists because KYC refresh work is usually treated like ticket residue instead of a governed operating surface.

The primitive here is simple:

- beneficial-owner evidence
- screening recency
- document expiry review
- unblock or hold posture

The implementation is intentionally synthetic and local-first, but the operating pattern is real: account-safe release decisions need an evidence lane that keeps ownership, screening, and final review posture aligned.
